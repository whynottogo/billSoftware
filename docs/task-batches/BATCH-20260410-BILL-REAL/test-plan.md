## 测试范围

- 覆盖页面：
  - `/user/bills/month`
  - `/user/bills/month/:month`
  - `/user/bills/year`
  - `/admin/users/:userId`
- 覆盖接口：
  - 用户端 bill 年度列表接口
  - 用户端 bill 月详情接口
  - 管理端单用户月/年 bill 详情接口
- 覆盖核心流程：
  - 用户端月账单列表真实渲染
  - 用户端月账单详情真实渲染
  - 用户端年账单真实渲染
  - 管理端查看单用户月/年账单详情
  - 管理端与用户端同口径对账

## 入口与前置条件

- 页面入口：
  - 用户端登录后进入 `/user/bills/month`
  - 管理端登录后进入 `/admin/users/:userId`
- 登录账号：
  - 用户：已启用测试用户
  - 管理员：已有可登录管理员账号
- 初始化数据：
  - 至少一个测试用户在 2 个以上月份内有 ledger 记录
  - 同一用户存在收入与支出分类，便于验证排行与分布
- 环境依赖：
  - 前后端本地可启动
  - 远端数据库可访问

## 执行顺序

1. 启动前后端并准备测试账号
2. 先做后端接口结构与关键字段验证
3. 验证用户端月账单列表与年账单页面
4. 验证用户端月账单详情页面
5. 验证管理端单用户账单详情页面
6. 对同一月份执行用户端与管理端口径对账
7. 归档截图、控制台、网络请求和接口摘要

## 验证策略

- 前端和联调任务使用 Codex tools 中的 Playwright 进行真实页面验证
- 关键统计接口同时保留 curl 或后端日志证据，便于核对口径
- 失败时必须先补充截图、控制台摘要、关键网络请求，再登记 Bug
- 实际执行中为用户 `e2e0410000402` 通过真实接口补齐了 2025-11 至 2026-04 的样本账单，用于验证跨月跨年统计

## 风险记录

- 风险 1：账单统计维度多，若后端未做共享聚合层，容易出现用户端与管理端口径不一致
- 风险 2：现有页面完全基于 mock 结构，真实接口返回字段若偏差较大，前端联调会出现连锁调整
